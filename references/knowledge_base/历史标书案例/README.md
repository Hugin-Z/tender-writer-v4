# 历史标书案例

## 用途

存放往期投标书的结构化解读,作为编制新标书时的**结构与话术参考**。包含中标和未中标两类:
- **中标案例**:学习其得分高的应答策略
- **未中标案例**:复盘扣分原因与遗漏点

## 摄入流程

1. 把往期标书原件(PDF / docx)复制到 `_inbox/` 目录
2. 在 Claude Code 中说"处理历史标书案例 inbox"
3. AI 调用 `extract_text.py` 提取文本,按 `案例schema.md` 结构化
4. 输出 `<时间戳>_<项目简称>.md`,frontmatter 中 `review_status=pending`
5. 用户人工 review 后,把 `review_status` 改为 `approved`
6. 原文件移动到 `_raw/`(加时间戳前缀)

## 三类型公司差异(关键)

| company_type | 是否允许进本目录 | 撰写新标书时的引用方式 |
|---|---|---|
| **own** | ✅ | 结构和话术可参考,**不能整段复制**;具体业绩/资质应放到 `assets/` |
| **partner** | ✅ | 结构可参考,**严禁**复制其未在 `assets/` 登记的具体素材 |
| **reference** | ✅(且**只能**进本目录) | **严禁**任何具体业绩/资质/人员/金额数据进入新标书正文,只能学习风格 |

## 目录约定

```
历史标书案例/
├── README.md           ← 本文件
├── 案例schema.md       ← 案例 .md 必须遵循的字段规范
├── _inbox/             ← 待摄入的原始 PDF/docx 临时区
├── _raw/               ← 摄入完成后的原文件归档(时间戳前缀)
└── <时间戳>_<项目简称>.md   ← 结构化后的案例文件
```
