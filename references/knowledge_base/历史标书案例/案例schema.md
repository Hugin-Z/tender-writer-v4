# 案例 schema(历史标书案例 .md 必须遵守的字段规范)

> 摄入流程会基于本 schema 把原始 PDF/docx 转写为结构化 markdown。
> AI 在执行摄入时**必须严格按本规范输出**,缺失字段标 `TODO:待人工确认`,严禁编造。

---

## 一、必填字段(frontmatter YAML)

```yaml
---
# === 公司归属与状态(所有 knowledge_base 文件通用)===
source_company: <公司 id,引用 companies.yaml>     # 必填
company_type: own | partner | reference            # 必填,直接决定能否被新标书参考
review_status: pending | approved                  # 摄入时默认 pending
source_file: <原始文件名,如:2024_xxx智慧城市投标书.pdf>  # 必填
ingest_date: <YYYY-MM-DD>                          # 必填,摄入当天日期

# === 案例业务元数据 ===
项目类型: <智慧城市 | 数字乡村 | 政务系统 | 教育信息化 | 医疗信息化 | ... >  # 必填
项目名称: <原标书项目全称>                         # 必填
甲方单位: <采购人,可脱敏为"某市某局">              # 必填
预算: <数字,单位万元>                              # 必填
项目周期: <如 12 个月>                              # 选填
结果: 中标 | 未中标 | 废标                          # 必填
评分: <如 85.6/100>                                 # 选填,有则填
排名: <如 第 1 名 / 第 3 名>                        # 选填
中标金额: <数字,单位万元>                          # 中标时必填
关键成功因素: []                                    # 中标时必填(列表)
失败原因: []                                        # 未中标或废标时必填(列表)
评委关注点: []                                      # 选填
---
```

---

## 二、必填字段(正文结构)

正文按以下章节组织,**每节字数不超过 500 字**,提炼性写作,严禁照搬原文段落:

```markdown
## 一、项目背景概要(≤ 200 字)
<项目所在地、政策背景、采购人主要诉求>

## 二、标书章节结构
<原标书的章节目录,层级缩进式列出>

## 三、评分项响应策略(≤ 500 字)
<原标书是如何针对评分办法逐项回应的,有哪些独到的策略>

## 四、可借鉴的话术风格(≤ 500 字)
<提炼出 3-5 条可复用的话术骨架,严禁原样复制具体业绩/资质/人员>

## 五、教训与启示(≤ 300 字)
<本案例能给后续投标的警示或经验>
```

---

## 三、引用原则

1. **所有事实必须能在原始文件中找到出处。** 严禁根据印象编造或外推。
2. 摘录原文时**必须改写**,不允许大段照抄(超过 50 字的原段必须改写)。
3. `company_type=reference` 的案例,**只能**抽取章节结构和写作策略,**严禁**记录其具体业绩/资质/人员/金额数据(即使原文有,也不要写进 .md)。
4. 不确定的字段一律标 `TODO:待人工确认`,**严禁猜测**。

---

## 四、脱敏规则

- **甲方单位**:可脱敏为"某市某局",视用途决定
- **本方人员姓名**:全部脱敏(如 "张某某")
- **身份证号 / 银行账号 / 手机号 / 邮箱**:必须脱敏(只保留前后各 2 位)
- **合同金额**:可保留(中标公告本来就是公开信息),但 `company_type=reference` 时仅保留量级(如"千万级"),不写具体数字
- **经办人 / 联系人姓名**:必须脱敏

---

## 五、未知字段处理

- 缺失字段一律填 `TODO:待人工确认`
- **严禁编造**项目名称、甲方单位、金额、时间、人员等任何事实
- frontmatter 中数值字段缺失时填 `null`,字符串字段缺失时填 `"TODO"`
- 摄入完成后,生成的 .md 顶部应有一个 `## TODO 清单` 临时区块,列出所有待人工确认的字段,review 通过后由人工删除该区块并把 `review_status` 改为 `approved`
