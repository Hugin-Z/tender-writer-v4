# 失败教训

## 用途

记录团队在投标过程中**踩过的坑**——废标原因、扣分点、被评委指出的硬伤、临场失误、装订/盖章错误等。AI 在阶段 5(合规终审)会扫描本目录,把过往教训作为额外的检查项加入合规清单。

> ⚠️ 本目录的每一条记录都是真金白银换来的。**请务必如实记录,不要美化、不要遮掩**。
>
> 一份诚实的失败教训库,比十份空泛的成功案例更值钱。

## 目录约定

每份 .md 都遵循 knowledge_base 通用 frontmatter:

```yaml
---
source_company: <来源公司 id>
company_type: own | partner | reference
review_status: pending | approved
source_file: <原始文件名,可为空,直接写复盘记录>
ingest_date: <YYYY-MM-DD>
---
```

文件名建议:
- `踩过的坑.md`(综合记录)
- `<年份>_<项目简称>_废标复盘.md`(单项目复盘)

## 必须记录的字段(每条教训)

- **发生时间**:具体到月
- **项目名称**:可脱敏
- **性质**:废标 / 扣分 / 被警告 / 临场失误
- **直接原因**:一句话说清
- **根本原因**:复盘到流程或制度层面
- **改进措施**:已经/即将采取的动作
- **复盘人**:可脱敏
