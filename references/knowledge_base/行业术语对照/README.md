# 行业术语对照

## 用途

记录各行业的标准术语、官方定义、常见误用。AI 在撰写正文时会扫描本目录,把通用术语替换为行业内的标准术语,避免出现"外行话"。

例如:
- "用户" → 在政务系统语境中应改为 "办事群众" 或 "申请人"
- "客户" → 在数字乡村语境中应改为 "村民" 或 "服务对象"
- "BS 架构" → 应改为 "B/S 架构"
- "云原生" → 应明确指代 K8s/容器化/微服务/不可变基础设施 等具体实践

## 目录约定

每份 .md 都遵循 knowledge_base 通用 frontmatter:

```yaml
---
source_company: <来源公司 id>
company_type: own | partner | reference
review_status: pending | approved
source_file: <原始文件名>
ingest_date: <YYYY-MM-DD>
---
```

文件名建议:`<行业>常用术语规范.md`,例如:
- `智慧城市常用术语规范.md`
- `数字乡村常用术语规范.md`
- `政务系统常用术语规范.md`
