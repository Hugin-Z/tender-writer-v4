# 政府项目投标合规规则与废标风险清单

> 本文档梳理政府类项目投标中**最常见的废标情形和合规风险点**,
> 用于阶段 1(招标文件解析)和阶段 5(合规终审)阶段作为检查依据。
> 注意:本清单为通用参考,**最终废标判定以招标文件原文和评标专家裁定为准**,
> 法务和标书经理的最终复核不可省略。

---

## 一、五大类常见废标原因

### A. 资格性废标(基础门槛)

| 编号 | 废标情形 | 检查方式 | 应对要点 |
|---|---|---|---|
| A1 | 投标人不具备招标文件要求的资质等级(如:要求 ISO27001 但未提供) | 对照招标文件资格要求逐项核对 | 提交前由商务岗逐项打钩 |
| A2 | 注册资金不达标 | 营业执照核对 | 招标文件公布即核对 |
| A3 | 近 X 年类似业绩不足 | 业绩证明材料核对 | 业绩合同复印件、用户证明、中标公告截图 |
| A4 | 关键人员资质过期或缺失(如项目经理 PMP/信息系统集成项目经理) | 人员证书复印件核对有效期 | 至少提前 1 个月核查证书有效期 |
| A5 | 未按要求缴纳投标保证金 | 银行回单 | 留足汇款时间,跨行汇款 T+1 |
| A6 | 未在规定时间内提交投标文件 | 提交时间记录 | 至少提前 1 小时到场 |
| A7 | 投标人为失信被执行人(信用中国查询) | 查询信用中国网站截图 | 投标前一周 + 当天各查一次 |
| A8 | 投标人存在重大违法记录 | 信用查询 | 同上 |

### B. 实质性响应废标(★/▲ 条款)

| 编号 | 废标情形 | 检查方式 | 应对要点 |
|---|---|---|---|
| B1 | 未对带 ★ 的条款做明确响应 | 全文检索 ★ 符号 | 每个 ★ 都要在应答表里有"完全响应"或"满足"字样 |
| B2 | 对 ★ 条款做了"部分响应"或"不响应" | 应答表逐项核对 | ★ 条款不允许有任何偏离 |
| B3 | 关键技术指标低于招标文件最低要求 | 技术参数对比表核对 | 用对比表逐项比对,不达标的项要么换型号要么放弃 |
| B4 | 工期承诺超出招标文件规定 | 工期承诺书核对 | 工期不能比招标文件长,可以短 |
| B5 | 质保期承诺低于招标文件要求 | 质保承诺书核对 | 质保期不能比招标文件短,可以长 |
| B6 | 技术方案与招标需求严重不符 | 技术方案章节与需求对比 | 用 scoring_matrix.csv 全程跟踪 |

### C. 报价废标(超预算 / 低于成本)

| 编号 | 废标情形 | 检查方式 | 应对要点 |
|---|---|---|---|
| C1 | 投标报价超过招标文件最高限价 | 报价单对照最高限价 | 报价前必须确认最高限价 |
| C2 | 投标报价低于成本(可能被认定为低于成本竞争) | 成本测算说明 | 必要时附成本说明 |
| C3 | 分项报价合计与总价不一致 | 报价单核对 | 用 Excel 公式自动计算总价 |
| C4 | 报价币种或单位错误 | 报价单核对 | 严格按招标文件要求 |
| C5 | 大写金额与小写金额不一致 | 报价单核对 | 双人复核 |
| C6 | 报价中有缺漏项 | 报价单与清单核对 | 用工程量清单逐项核对 |

### D. 签字盖章 / 装订格式废标

| 编号 | 废标情形 | 检查方式 | 应对要点 |
|---|---|---|---|
| D1 | 法定代表人或授权代表未签字 | 全文翻阅签字处 | 用清单逐页核对 |
| D2 | 未加盖投标人公章 | 全文翻阅盖章处 | 同上 |
| D3 | 关键页缺骑缝章 | 翻阅装订面 | 装订前盖好骑缝章 |
| D4 | 投标文件未按要求密封 | 密封情况核对 | 提交前最后一道工序 |
| D5 | 投标文件份数不足(正本/副本/电子版) | 数量核对 | 至少多备一份 |
| D6 | 装订方式不符合要求(胶装/线装/活页夹) | 装订成品核对 | 提前确认装订厂能做指定工艺 |
| D7 | 投标文件封面格式不符合要求 | 封面核对 | 严格按招标文件附件中的封面模板 |

### E. 格式 / 内容低级错误

| 编号 | 废标情形 | 检查方式 | 应对要点 |
|---|---|---|---|
| E1 | 投标人名称与营业执照不一致 | 全文搜索投标人名称 | 用全文替换确保一致 |
| E2 | 项目名称写错或与招标文件不一致 | 全文搜索项目名称 | 复制粘贴自招标文件 |
| E3 | 出现其他投标人或其他项目的名称(复用模板未清理) | 全文检索竞争对手名称、其他项目名称 | 改卷前必须做"模板残留"检查 |
| E4 | 字体、字号、行距明显与招标文件要求不符 | 抽查若干段落 | 用 docx_builder.py 统一应用规范 |
| E5 | 目录页码与正文不一致 | 目录核对 | 自动目录,不要手写 |
| E6 | 出现明显的错别字或事实错误(如数据矛盾) | 通读全文 | 至少两人交叉校对 |

## 二、合规检查清单(checklist)

> 本清单可作为 `compliance_check.py` 的检查项参考,也可在交付前由人工逐项打钩。

### 阶段性检查(每个阶段都做)

- [ ] tender_brief.md 中"实质性响应条款"和"废标条款"两节是否完整提取?
- [ ] scoring_matrix.csv 中是否有评分项被标注为"风险:漏答"?
- [ ] outline.md 中是否有评分项未被任何章节覆盖?

### 资格性自检

- [ ] 营业执照有效期、注册资金达标
- [ ] 招标文件要求的所有资质证书复印件齐备且在有效期内
- [ ] 信用中国、信用地方网站查询截图(投标当天)
- [ ] 关键人员的身份证、社保证明、职业资质证书齐备
- [ ] 类似业绩证明材料齐备(合同、用户证明、中标公告任选)
- [ ] 投标保证金已缴纳,银行回单已附

### 实质性响应自检

- [ ] 全文搜索 ★ 符号,每个 ★ 都已在应答表中"完全响应"
- [ ] 全文搜索 ▲ 符号(若有),响应情况已说明
- [ ] 技术参数对比表中每一项都标注"满足"或"优于"
- [ ] 工期承诺 ≤ 招标文件要求工期
- [ ] 质保期承诺 ≥ 招标文件要求质保期

### 报价合规自检(虽然技术标不涉及报价,但若混编需检查)

- [ ] 总价 ≤ 最高限价
- [ ] 大小写金额一致
- [ ] 分项报价合计 = 总价
- [ ] 不存在缺漏项

### 签字盖章自检

- [ ] 投标函页:法定代表人/授权代表签字 + 公章
- [ ] 法人授权委托书:法定代表人签字 + 受托人签字 + 公章
- [ ] 资质证书复印件:每页加盖公章
- [ ] 技术方案章节首页:公章
- [ ] 骑缝章:正本和副本均盖
- [ ] 封面:项目名称、投标人名称、日期完整且公章齐备

### 装订与提交自检

- [ ] 正本份数符合要求(通常 1 份)
- [ ] 副本份数符合要求(通常 4-6 份)
- [ ] 电子版(U 盘 / 光盘)份数符合要求
- [ ] 密封袋上的项目名称、投标人名称、密封章齐备
- [ ] 投标文件提交时间留出至少 1 小时缓冲

### 模板残留自检

- [ ] 全文搜索"xxx 公司"等占位符,确认无残留
- [ ] 全文搜索其他投标项目名称、其他客户名称,确认无残留
- [ ] 全文搜索其他投标人/竞争对手名称,确认无残留
- [ ] 页眉页脚中的"项目名称"和"投标人名称"已替换

## 三、特别提醒

1. **★ 条款是高压线**:任何 ★ 条款响应不到位都是直接废标,没有商量余地。建议在 scoring_matrix.csv 中给所有 ★ 条款单独着色或标记,合规终审时优先核查。

2. **信用查询必须留证**:不仅是查一下没事就行,而是要把"信用中国 + 信用地方"两个网站的查询页面**截图存档**作为投标文件附件。

3. **签字盖章是低级错误**:每年都有公司栽在"封面忘记盖章""授权委托书忘记签字"上。建议建立"盖章 checklist",提交前由商务岗双人复核。

4. **模板残留是隐藏杀手**:用旧标书改新标书时,容易残留旧客户名称、旧项目名称甚至竞争对手名称。务必做全文搜索清理。

5. **临场应变**:如果开标当天才发现某项资格不达标,**不要硬投**。废标是公开记录,会影响信用评级和后续投标资格。
