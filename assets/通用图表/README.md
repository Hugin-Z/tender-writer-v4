# 通用图表

## 用途

存放可在多个标书中复用的通用图表(架构图、流程图、甘特图、组织架构图、技术路线图等)的索引和说明,供阶段 4 撰写正文时引用。

> 💡 本目录**不存放图片本身**(图片体积大,放在 `_raw/` 下归档),只存放结构化的**说明 .md**(图表标题、类型、适用场景、引用路径)。

## 公司归属规则

按 `<company_id>/` 子目录隔离:
- **own**:我方主体的图表,可直接引用
- **partner**:合作方提供的图表,**仅联合体投标时**可引用,且**必须在图注中标注来源**
- **reference**:**严禁**出现在本目录(竞品的架构图绝对不能用)

## 摄入流程

1. 把图片(PNG/JPG/SVG/Visio)和说明文档放入 `<company_id>/_inbox/`
2. 在 Claude Code 中说"处理 <company_id> 的图表 inbox"
3. AI 按 `图表schema.md` 结构化,生成 `<图表标题>.md`
4. 同时追加索引到 `<company_id>/图表索引.md`
5. 原图移动到 `_raw/`(加时间戳前缀)
6. 用户人工 review,把 `review_status` 改为 `approved`

## 引用注意

- 撰写标书时,**架构图必须根据本项目业务模块名称重新调整**,严禁原样复用其他项目的架构图(评委一眼能看出"通用 PPT")
- 流程图、甘特图、组织架构图同理
- 通用图表只是**模板和素材**,不是成品
