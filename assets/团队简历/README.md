# 团队简历

## 用途

存放公司技术与管理团队成员的结构化简历(姓名脱敏、岗位、证书、业绩等),供阶段 4 撰写"项目组织架构""关键岗位人员"章节时调用。

## 公司归属规则

按 `<company_id>/` 子目录隔离:
- **own**:我方主体团队,可直接引用
- **partner**:合作方/分包方人员,**联合体投标时**可引用,且**必须在文中明确标注合作方公司名称**
- **reference**:**严禁**出现在本目录

## 关键合规点(阶段 5 终审协同)

阶段 5 合规终审会扫描 `<own_company_id>/简历索引.csv` 中所有 `approved` 简历,**检查证书有效期**:
- 过期证书 → 标红
- 临期(< 30 天) → 标黄并提示"投标前是否重新核查"
- 关键岗位(项目经理 / 安全负责人)缺少招标文件要求的资质 → 标红

## 摄入流程

1. 把简历(docx / PDF)放入 `<company_id>/_inbox/`
2. 在 Claude Code 中说"处理 <company_id> 的简历 inbox"
3. AI 调用 `extract_text.py` 提取文字,按 `简历schema.md` 结构化
4. 输出 `<姓名脱敏>.md`,frontmatter 中 `review_status=pending`
5. 同时追加一行到 `<company_id>/简历索引.csv`
6. 原文件移动到 `_raw/`(加时间戳前缀)
7. 用户人工 review 并补全 TODO 字段,把 `review_status` 改为 `approved`

## 隐私与脱敏强约束

- 摄入时**必须**对身份证号、手机号、邮箱、家庭住址做脱敏
- 简历正文中**严禁**出现身份证号完整数字
- **姓名按公司规范脱敏**(如 "张某某" 或 "张 X 强"),具体保留多少由公司决定
