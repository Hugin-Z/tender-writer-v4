# own_default · 团队简历目录

## 这是什么

这是一个**空壳占位目录**,对应 `companies.yaml` 中 `id: own_default` 的占位条目(`status: placeholder`)。

## 首次使用

**不要直接往这个目录放真实简历(身份证/职称证/社保证明等)**。请先走"新增公司"流程登记你方实际投标主体,再把该主体的真实团队简历放到对应 `assets/团队简历/<你的 company_id>/` 目录。

新增公司的命令:

```bash
./run_script.bat add_company.py "你公司全称" own --alias "你公司简称"
```

## 简历素材摄入

简历放到 `_inbox/` 后,用以下命令自动抽取要素并登记:

```bash
./run_script.bat ingest_assets.py 简历 <你的 company_id>
```

## 简历索引

该目录已带一个空骨架 `简历索引.csv`,保留不动。新增公司后,真实简历索引会写入 `assets/团队简历/<你的 company_id>/简历索引.csv`。

## 合规与隐私提醒

- 包含个人信息(身份证、社保号等)的材料属**敏感素材**,入库前请确认已获取本人授权
- 真实简历证书在 `_inbox/` → `_raw/` 流转过程中会被时间戳重命名并归档,`_raw/` 内容**不进 git**(见根 `.gitignore`)
- 招标文件如无"提供团队成员名单"硬要求,可以不上传简历证书原件,仅在响应文件中列团队清单
