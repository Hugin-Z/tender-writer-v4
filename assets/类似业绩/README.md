# 类似业绩

## 用途

存放公司过往项目的结构化业绩记录(项目名称、甲方、金额、技术、效益等),供阶段 4 撰写"类似业绩"和"项目案例"章节时调用。

## 公司归属规则

按 `<company_id>/` 子目录隔离:
- **own**:我方主体业绩,可直接引用
- **partner**:合作方业绩,**仅联合体投标时**可引用,且**必须在文中标注"由 [partner 公司名] 提供"**
- **reference**:**严禁**出现在本目录(竞品案例只能进 `references/knowledge_base/`,且严禁写入标书正文)

## 筛选优先级(给 AI 阶段 4 撰写时使用)

撰写新标书的"类似业绩"章节时,从 `<own_company_id>/业绩列表.csv` 中按以下优先级筛选 3-5 条最相关的业绩深度展开:

1. **同行业**(政务/智慧城市/数字乡村/...)
2. **同规模**(预算量级在 ±50% 范围内)
3. **同地区**(同省 > 同区域 > 全国)
4. **同类技术**(微服务/中台/AI/物联网/...)

## 摄入流程

1. 把往期项目的相关材料(中标通知书、合同、验收报告、宣传稿)放入 `<company_id>/_inbox/`
2. 在 Claude Code 中说"处理 <company_id> 的业绩 inbox"
3. AI 调用 `extract_text.py` 提取文字,按 `业绩schema.md` 结构化
4. 输出 `<项目简称>.md`,frontmatter 中 `review_status=pending`
5. 同时追加一行到 `<company_id>/业绩列表.csv`
6. 原文件移动到 `_raw/`(加时间戳前缀)
7. 用户人工 review 并补全 TODO 字段,把 `review_status` 改为 `approved`
