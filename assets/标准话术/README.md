# 标准话术

## 用途

存放公司沉淀的高质量话术片段(章节开头骨架、技术介绍模板、安全方案模板、培训方案模板等),供阶段 4 撰写正文时**作为骨架引用**。

## 与 references/phrase_library.md 的区别

| 维度 | `references/phrase_library.md` | `assets/标准话术/` |
|---|---|---|
| 性质 | 通用骨架(无公司归属) | 公司沉淀的话术资产(按公司隔离) |
| review 流程 | 无 | 有 pending → approved 流程 |
| 引用方式 | 阶段 4 撰写时作为通用参考 | 阶段 4 撰写时作为优先引用源 |
| 改写要求 | 必须本地化改写 | **必须本地化改写**(同样不允许原样复制) |

> ⚠️ **标准话术必须本地化改写,严禁原样复制粘贴进新标书**。即使是 own 类型,也不允许"复制粘贴大法"。话术库提供的是"骨架",不是"成品"。

## 公司归属规则

按 `<company_id>/` 子目录隔离:
- **own**:可作为骨架引用,**必须改写后**进入新标书
- **partner**:**仅联合体投标时**可引用,且必须改写并标注来源
- **reference**:**严禁**出现在本目录

## 摄入流程

1. 把高质量话术片段(从中标项目中提炼)整理成 docx / md 放入 `<company_id>/_inbox/`
2. 在 Claude Code 中说"处理 <company_id> 的话术 inbox"
3. AI 按 `话术schema.md` 拆解为单个话术 .md
4. 同时追加索引到 `<company_id>/话术索引.md`
5. 用户人工 review,把 `review_status` 改为 `approved`

## 沉淀来源建议

- 中标项目的金句、被评委点名表扬的段落
- 被采购方反馈"写得很到位"的措辞
- 团队内部认为"以后可以复用"的好句子
