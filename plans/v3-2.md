## 3. 测试框架与目录设计

### 3.1 测试风格(对齐既有 test_budget_parsing.py)

- **stdlib only**:不引入 pytest/unittest 依赖。每个 `test_*.py` 是独立可执行脚本,`main()` 返回退出码(0=全过,1=有失败)。
- **fixture 形式**:CASES 列表(`(input, expected, note)` 三元组)+ 主循环逐项断言,失败打印 PASS/FAIL 行 + 末尾汇总。
- **运行入口**:`./run_script.bat tests/test_xxx.py`(走 .venv 里的 python)。
- **重写 vs import**:对于纯解析函数(如 `parse_money_in_yuan`),允许 `from scripts.xxx import` 直接用;对于带 IO 的函数(如 `parse_budget` 在 budget_parsing 里被重写过)仍可重写来保证 fixture 独立。**默认 import**,偏向真实函数;只有解析逻辑高度内嵌于 main() 的才重写。

### 4.7 tests/run_all.py(基础设施)

**职责**:发现 `scripts/tests/test_*.py`,逐个调用其 `main()`,汇总退出码。

**实现要点**(stdlib):

```python
import importlib.util, sys
from pathlib import Path

def discover_tests():
    return sorted(Path(__file__).parent.glob("test_*.py"))

def run_one(test_path):
    spec = importlib.util.spec_from_file_location(test_path.stem, test_path)
    mod = importlib.util.module_from_spec(spec)
    spec.loader.exec_module(mod)
    return mod.main()

def main():
    results = []
    for tp in discover_tests():
        print(f"\n{'='*60}\n运行: {tp.name}\n{'='*60}")
        rc = run_one(tp)
        results.append((tp.name, rc))
    print(f"\n{'='*60}\n总览:")
    fails = sum(1 for _, rc in results if rc != 0)
    for name, rc in results:
        print(f"  [{'PASS' if rc==0 else 'FAIL'}] {name}")
    return 1 if fails else 0

if __name__ == "__main__":
    sys.exit(main())
```
