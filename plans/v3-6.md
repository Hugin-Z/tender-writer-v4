## 0. Phase 0 关键发现

### 0.1 字符密度实测

用 `pdfplumber.extract_text()` 实测样本：

| 样本 | 页数 | 总字符 | 平均字/页 | 单页最低 | 单页最高 | <50 字页数 |
|---|---|---|---|---|---|---|
| 国家自然科学基金（文字版） | 59 | 27,492 | **466** | 31 | 993 | 1（封面）|
| 深圳国资委（文字版） | 55 | 38,415 | **698** | 36 | 1331 | 2 |
| 合成扫描版（PIL→PDF 4 页） | 4 | **0** | **0** | 0 | 0 | 4 |

合成扫描版生成方式：PIL 渲染中文文字到 850×1100 白底图，多页 `image.save('out.pdf', save_all=True)`。`pdfplumber.extract_text()` 在这种 PDF 上**完全无法提取**（图片 PDF 无文字流），返回空字符串。

### 0.2 阈值决策

**D0**：判定口径 = **全文平均字/页 < 50 → 扫描版**。

- 文字版下限 466，扫描版顶 0，**差距 9× 以上**，50 字/页阈值非常稳
- **不能用逐页判定** —— 正常 PDF 封面页 31~47 字会误判
- 兜底：全文总字符 < 100 也判扫描版（防 1-2 页文档边界）

**计划文件原写"50 字/页"是逐页阈值，本 plan 改为全文平均**——这是 Phase 0 实测后的修正。

### 0.3 边界场景识别

| 场景 | 平均字/页预估 | 是否误报 |
|---|---|---|
| 极简纯封面 PDF（1 页 30 字）| 30 | **会误报**（< 50）|
| 极简 PDF（2 页 总 200 字 = 100/页）| 100 | 不误报 |
| 加密 PDF | 0 | 误报为扫描版（实际是别的问题）|
| 含大量图但有文字的 PDF | 通常 ≥ 100 | 不误报 |

**极简纯封面 PDF 误报**是真实风险，但实际意义小（1 页 30 字的 PDF 无法当招标文件解析）。设计上接受这个边界，让脚本提示"建议人工核实是否扫描版"，由用户决定走 fallback 还是不走。**本计划不为这个边界专门做 fixture 验证**，只在风险表 + Q4 文档化。

## 2. 实现设计

### 2.1 read_pdf 函数扩展

```python
def read_pdf(path: Path, ocr_fallback_txt: Path | None = None) -> str:
    """读取 PDF 文件，返回纯文本。

    扫描版检测（V3-6）：
    - 提取后全文平均字/页 < 50 视为扫描版
    - 有 ocr_fallback_txt 参数 → 读取 txt 作为 raw_text 返回
    - 无参数 → 抛 ScannedPdfDetected，main 捕获后友好报错退出
    """
    try:
        import pdfplumber
    except ImportError:
        ...

    pages_text = []
    with pdfplumber.open(str(path)) as pdf:
        n_pages = len(pdf.pages)
        for page in pdf.pages:
            text = page.extract_text() or ""
            pages_text.append(text)

    full_text = "\n\n".join(pages_text)
    total_chars = sum(len(t) for t in pages_text)
    avg_per_page = total_chars / n_pages if n_pages else 0

    is_scanned = (avg_per_page < 50) or (total_chars < 100)

    if is_scanned and ocr_fallback_txt is None:
        raise ScannedPdfDetected(
            n_pages=n_pages, total_chars=total_chars,
            avg_per_page=avg_per_page, pdf_path=path,
        )

    if is_scanned and ocr_fallback_txt is not None:
        if not ocr_fallback_txt.exists():
            print(f"[错误] --ocr-text 指定的文件不存在：{ocr_fallback_txt}",
                  file=sys.stderr)
            sys.exit(1)
        return ocr_fallback_txt.read_text(encoding="utf-8")

    return full_text
```

新异常类（顶层）：

```python
class ScannedPdfDetected(Exception):
    """扫描版 PDF 检测命中，无 fallback 时抛出。"""
    def __init__(self, n_pages, total_chars, avg_per_page, pdf_path):
        self.n_pages = n_pages
        self.total_chars = total_chars
        self.avg_per_page = avg_per_page
        self.pdf_path = pdf_path
        super().__init__(
            f"PDF 疑似扫描版：{n_pages} 页 / {total_chars} 字符 / "
            f"平均 {avg_per_page:.1f} 字/页（< 50 阈值）"
        )
```

### 2.3 用户友好提示输出（核心 UX）

`print_scanned_pdf_help(e)` 输出（中文，对齐 SKILL.md 风格）：

```
[错误] 检测到疑似扫描版 PDF：
  文件：<pdf_path>
  全文 <n_pages> 页 / <total_chars> 字符 / 平均 <avg_per_page:.1f> 字/页
  阈值：50 字/页（实测文字版招标 PDF 平均 466~700+ 字/页）

扫描版 PDF 没有内嵌文字流，pdfplumber 无法提取文本。请按以下三条路径之一处理：

[路径 1] 本地 OCR（推荐技术用户）
  pip install ocrmypdf
  装 tesseract（含 chi_sim 中文包）
  ocrmypdf -l chi_sim "<原 PDF>" "<输出 PDF>"
  然后用输出 PDF 重跑 parse_tender

[路径 2] 外部 OCR / AI 多模态服务
  上传扫描版 PDF 到豆包/通义/Claude/GPT 等多模态 AI
  让其转出纯文本（注意核对 ★/▲ 条款逐字）
  存为 <pdf_name>.txt，重跑：
    parse_tender.py "<原 PDF>" --ocr-text "<pdf_name>.txt"

[路径 3] 找原版文字 PDF
  联系采购方 / 招标代理获取从 Word 直接导出的文字版 PDF
  最稳妥，不引入 OCR 错字风险

详见 docs/FAQ.md Q4。

[退出] 扫描版 PDF 处理需用户介入，parse_tender 暂停。
```

**为什么不打印 `--ocr-text` 自动重试命令**：因为路径 2 完成后用户已知道用 fallback flag，无需脚本帮记忆；路径 1/3 完成后用户应该用产出的新 PDF 直接跑（不需要 fallback）。

### 2.4 误报控制

- **检测阈值是 hardcode 50** —— 实测有 9× 安全余量，不需用户调
- **不引入 `--scan-threshold` flag** —— V3 范围内不需要；如未来发现误报多，登记 V4
- **不引入 `--force-text-mode` flag** —— 用户若坚信不是扫描版，直接走 `--ocr-text` 用空 txt 也能绕过（虽然下游会报字段缺失，但 escape hatch 存在）
- **极简 PDF 误报接受** —— 30 字/页的 PDF 实际无法当招标文件用，提示"疑似扫描版"是合理引导
