# _inbox_unsorted · 待分类材料临时收件箱

## 用途

当你拿到一份材料但**不确定该归到哪一类、哪家公司**时,直接把它丢到本目录,然后交给 AI 执行**分类 triage 流程**(详见 SKILL.md 第十章)。

典型场景:
- 一份往期中标的完整投标包,内部同时包含历史案例、业绩信息、团队简历、资质证书 —— 需要 AI 识别并拆分到多个目标位置
- 一份从合作方拿到的混合材料包,既有 own 类素材也有 partner 类素材
- 一份不确定是 own 还是 partner 来源的资料,需要先确认归属再分类
- 一份下载自竞品官网的公开案例集,需要 AI 识别为 reference 类型并送入 knowledge_base/

## 怎么用

1. 把待分类的文件(PDF / docx)复制或拖放到本目录(`tender-writer/_inbox_unsorted/`)。**严禁**把当前正在投标项目的招标文件本身丢到这里——招标文件属于"项目数据",不应进入素材库。
2. 在工作目录下启动 Claude Code(或其他 AI 工具),说一句:
   - "处理 _inbox_unsorted"
   - "我有一堆材料不知道怎么分类"
3. AI 会扫描每份文件,生成**分类建议**(目标类别 + 目标公司 + 判断理由 + 目标路径),然后**等待你逐条确认**。
4. 你确认后,AI 会把文件副本放入对应的 `assets/<类别>/<公司>/_inbox/` 或 `references/knowledge_base/历史标书案例/_inbox/`,并自动触发后续的素材摄入流程。原文件会被移动到 `_inbox_unsorted/_raw/` 并加上时间戳前缀。

## 严格约束(AI 必须遵守)

- ❌ **严禁未经用户确认就自动分类**
- ❌ **严禁默认归为 own**(必须明确询问公司归属)
- ❌ **严禁把 reference 类型材料写入 assets/**(reference 只能进 knowledge_base/)
- ❌ **严禁把当前项目的招标文件、甲方提供的材料摄入到 knowledge_base 或 assets**
- ✅ 对公司归属有任何不确定时,必须主动询问用户

## 目录结构

```
_inbox_unsorted/
├── README.md       ← 你正在看的这份说明
├── _raw/           ← AI 处理完成后,原文件会移动到这里(加时间戳前缀)
└── (待分类的文件就放在本目录下)
```
